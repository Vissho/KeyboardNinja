A smart pointer is a memory indirection idiom that is widely used in programming in high-level languages.
These are usually used with objects that have special operations <<increase reference count>>.
Most often, such objects are inherited from a special class or interface.
The object must store inside itself a non-negative integer, which means the number of external pointers referring to this object.
When assigning the address of a new object to a pointer, the pointer calls a method on the object.
If the pointer previously referred to another object, then the "release" method of the old object is called first.
When deleting a pointer (its exit from the scope or destruction of the object whose field it was).
If this pointer refers to an existing object, the pointer calls the object's "release" method.
